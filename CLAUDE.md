## Quick Facts

- Game: **BLIP** — "You are the thing on the radar." Player: **CONTACT-47**. Route system: **Signal Nodes**. Villain/system: **The Interpretation Engine**. Story heart: **The Five Signal Scouts** (Will, Chip, Henry, Cameron, Danny).
- Current playable route: Miller Surface (`surface-z1`) → Motel Circuit (`circuit-z2`) → Chagrin Falls town/stadium connector (`town-z3`) → Patterson's Orchard maze (`maze-z4`) → Signal Storm (`anomaly-01`). These are separate top-down arena maps connected by fast breach handoffs with shared save/runtime state, not a seamless open map yet. Preserve the Five Signal Scouts and classification theme while polishing the route.
- Current combat foundation: Pulse Carbine, Arc Blade and Recall Disc with fast switching. Pulse is faster/cleaner at range with brighter laser-slug projectile trails, Arc has a wider close-range burst plus always-on parry shockwave and a heavier slash wedge, and Recall Disc has a consistent return trail. A centralized damage-family system maps Pulse/Arc/Kinetic/Blast affinities for all enemies; CIPHER, GRAVITON, UNDERTOW, DECOY and DORMANT are live tactical roles with WEAK/RESIST feedback and per-level rosters, but all five need final custom HD art. Important weapon pickups now show role cards; Arc pickups advertise PARRY/CLOSE and AI personas may prefer Arc against visible wardens/splitters/turrets. Boost is live on hold SHIFT/RB/touch Boost: it drains a boost meter, regenerates after release, has an explicit BOOST meter in the top-down HUD, leaves restrained behind-player echoes plus a capped fading cyan hover-fire ground trail, and keeps scanner crossing / brief i-frame counterplay. Miller now has the first red corrupted Boost washout crossings; they block normal walking and allow held-Boost crossing, but are not yet rolled out globally. Keep "dash" wording only for legacy internal ids. Region goals/rewards are named in HUD/telemetry. Orchard has the first playable Gravity Well/raised-ridge traversal beat, Crop Circle completion is gated on using it, and the HUD explicitly guides Gravity Well first then Crop Circle. Signal Tubes, Phase Doors, Signal Rails, deeper stealth, underground areas and larger puzzle chains remain scoped future work, not broad expansion.
- Authoritative backlog: use `MASTER_GAME_BACKLOG.md` plus Command Center `BUILD TODO` before using old prompts or summaries. It reconciles implemented, partial, planned, deferred, cut and decision-needed work.
- Current layout focus: schematic-first expanded layouts plus the first named-area gameplay/content pass are implemented across Miller, Motel, Town, Orchard and Signal Storm. All five are larger without changing actor/prop scale, route/scanner/gravity helper coordinates are data-driven, and the default gameplay camera is pulled back for better readable scale. A shared HD terrain environment-depth pass now adds biome-specific ground wear, medium/tall silhouettes, foreground framing, roof/parapet/service detail, local dressing around route-beacon names and data-driven elevation zones with stronger follow-camera offset/zoom so labels like Main Street, Pool Courtyard, Orchard Rows, Raised Ridge and Classifier Core have visual context and less flatness. `npm run qa:maps` now enforces marker walkability, route distances, purposeful optional branches, reward/event density, scanner label clarity, route-sign label crowding, scanner/Gravity Well/finale hooks and obvious empty-branch risk. `npm run qa:route` boots every route region and verifies HD rendering, objective/reward copy, route-open hints, pressure clearing, Motel scanner identity/offline cleanup, Ghost Check-In stealth-bonus gating, Orchard Gravity Well gating and Signal Storm finale copy. Remaining "feel review" is subjective only: travel feel, visual taste, cover quality, named-area visual identity and reward excitement.
- Map planning source of truth: use `MAP_SCHEMATICS.md` and the generated SVG/PNG files under `docs/map-schematics/` for top-down route planning. Regenerate them with `node scripts/generate-map-schematics.mjs` after arena coordinate changes and run `npm run qa:maps`.
- Testing vocabulary: `npm run typecheck`, `npm run build`, and Playwright suites are hard pass/fail deploy gates. Run separate Playwright suites sequentially because the shared `webServer` uses one preview port; the standalone `npm run ai:campaign` runner is safe to isolate because it uses process-specific strict preview ports. `npm run ai:campaign` is a heuristic playtest tool; its completion percentages are evidence for design priorities, not proof that the game is "passing." Latest public AI JSON is `first-three-route-regression-v8`: 6 Miller→Motel→Town attempts, 6/6 completions, 0 deaths, 0 soft-lock-risk stalls after nearest-useful route-marker targeting, objective-marker stuck recovery and fair expanded-route timing. Reports include per-run time budgets, objective target labels, decision traces and stall samples. Orchard-focused evidence remains `orchard-ai-gravity-priority-v1`: 6/6 completions with 0 soft locks/deaths. Extended/full-route AI campaigns remain pending.
- Camera shake rule: gameplay camera shake is reserved for CONTACT-47 taking damage. Do not add shake to routine shooting, impacts, kills, rewards, scanner alerts, overdrive, traversal, or UI reward flashes.
- Settings filters: CRT scanlines default OFF; CONTACT-47 personal aura defaults OFF and can be toggled in Settings; the screen-filter dropdown and 0-100 intensity slider apply to both the title and the active top-down world camera. Gameplay filters are automatically capped by `FILTER_GAME_STRENGTH` so the world stays readable; DOM/HUD overlays are not shader-filtered.
- Notification/reward UI truth: combat toasts are an activity feed capped at two with shorter routine dwell; important route/alert toasts stay longer; major weapon/reward/trophy reveals are centered blocking read-and-continue modals that pause gameplay and queue one at a time, with completion bursts batched into grouped cards where possible. Random health drops must not carry weapon ids; only `ptype: weapon` pickups should set `wid`.
- Route/readability truth: objective arrows target the nearest useful unvisited route marker so players are not sent backward to skipped signs. Field-event reward markers should state purpose (`CACHE`, `RECOVERY`, `SCOUT TECH`, `WEAPON`, `OVERDRIVE`) rather than generic `SCAN`/`POWER`, and Motel HUD copy should say scanners are offline instead of gate wording. Motel route signs use place/task names like `SCANNER CORE`; scanner labels must end in `SCANNER`, active beams should read red, and scanner devices should never look like unlabeled random props. Motel scanner copy should teach holding Boost, not tapping an old blink.
- Top-down pixel action game. Internal resolution 480×270, pixel-perfect upscale, with selected arenas using the higher-detail top-down render path. Phaser 3 + Arcade physics + Vite 5 (Node 18 pin) + TypeScript.
- Phaser Arcade physics is the active top-down collision/movement engine, not legacy side-scroller content. Do not remove it as "old arcade" code; keep visual sprite scale decoupled from world-space hitboxes through `ActorRig.collisionPx`.
- All art is procedural (generated textures). All tuning constants in `src/game/config.ts`. All content data in `src/game/data/`.
- Save: localStorage `blip_save_v1` (migrates legacy `beamline_save_v1` if found).
- Commands: `npm run dev` · `npm run typecheck` · `npm run build` · `npm run preview` · `npm run test:e2e` · `npm run qa:playtest` (fun-loop storyboard → `qa-reports/`) · `npm run qa:full` · `npm run qa:loop`
- No BEAMLINE branding anywhere user-facing (legacy save migration only).
